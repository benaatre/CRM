# CLAUDE.md — دليل العمل على «مشاريع السلطان CRM»

نظام إدارة مبيعات عقاري (CRM) لشركة **مشاريع السلطان** بالسعودية. عربي بالكامل، RTL، ثيم أوبسيديان داكن. هذا الملف يوجّه أي مساعد (Claude Code) يشتغل على الريبو.

## المرجع البصري (مهم)

التصميم في `docs/design-reference/` هو **مرجع بصري فقط** (ألوان + تخطيط + مكوّنات + أحجام بطاقات) — **لا يُنفّذ كصفحة HTML ثابتة**. نبني كل شي كمكوّنات React/Next حقيقية مربوطة بقاعدة البيانات:

- `design-brief.md` — البريف الكامل (الشاشات، المكوّنات، الإحساس، الألوان بالضبط).
- `schema.md` — أعمدة قاعدة العملاء والقيم المسموحة ومراحل خط البيع.
- `message-templates.md` — قوالب رسائل واتساب بلهجة سعودية.
- `sultan-crm-standalone.html` — نموذج التصميم القديم (للاستئناس البصري).

## التقنيات

- **Next.js 15** (App Router) + **TypeScript** + **Tailwind v4** + **shadcn/ui**.
- **Prisma** + **PostgreSQL**.
- **Auth.js (NextAuth v5)** — أدوار `owner / admin / employee`. الموظف يدخل برمز **PIN**.
- المساعد الذكي عبر **route handler** يستخدم **Anthropic API** بمفتاح في `.env` فقط (لا يُكشف للمتصفح).

## القواعد الثابتة

1. **اللغة:** كل النصوص بالعربي بلهجة سعودية طبيعية (مو فصحى جامدة ولا مصرية). مثال: «وزّعهم الآن»، «لسة ما دخلت».
2. **الاتجاه:** RTL في كل مكان (`dir="rtl"` على `<html>`).
3. **الهوية:** ألوان أوبسيديان (انظر `globals.css`)، خط IBM Plex Sans Arabic للواجهة، Reem Kufi للشعار.
4. **الصلاحيات على الخادم:** كل query/action يتحقق من دور المستخدم على الخادم — لا نعتمد على إخفاء عناصر الواجهة فقط. الموظف يشوف عملاءه فقط؛ المدير/المالك يشوفون الكل.
5. **السعودي:** العملة `ر.س`، اختصار الأرقام (٦٩٠ك / ١.٢م)، رقم ترخيص فال (REGA) بالواجهة، أسماء بنوك سعودية.
6. **لا تخترع بيانات:** لا أسعار/مواقع/عوائد غير موجودة في البيانات. أي إعلان يحمل سطر ترخيص فال.

## بنية المشروع

```
src/
  app/            # صفحات App Router + route handlers
  lib/
    prisma.ts     # عميل Prisma (singleton)
    utils.ts      # cn() لدمج أصناف Tailwind
  components/      # مكوّنات shadcn/ui + مكوّناتنا
prisma/
  schema.prisma   # المخطط
docs/
  design-reference/  # المرجع البصري (لا يُنفّذ كـ HTML)
sultan-crm-nextjs-spec.md  # المواصفات + خارطة المراحل (Milestones)
```

## أوامر شائعة

| الأمر | الوظيفة |
|------|---------|
| `npm run dev` | تشغيل التطوير |
| `npm run build` | توليد Prisma + بناء الإنتاج |
| `npm start` | تشغيل الإنتاج |
| `npm run db:migrate` | ترحيل تطويري (`prisma migrate dev`) |
| `npm run db:deploy` | ترحيل الإنتاج (`prisma migrate deploy`) |
| `npm run db:studio` | متصفح قاعدة البيانات |

## ملاحظات

- لا ترفع `.env` (مُستثنى). استخدم `.env.example` كمرجع.
- المخطط الحالي «أولي» (المرحلة ١) — يتوسّع مع المراحل (انظر ملف المواصفات).
- راجع `sultan-crm-nextjs-spec.md` لخارطة المراحل قبل بدء أي مرحلة جديدة.
