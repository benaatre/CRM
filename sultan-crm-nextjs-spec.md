# مواصفات «مشاريع السلطان CRM» — تطبيق Next.js إنتاجي

> هذا الملف أُلّف من **بريف التصميم** (`docs/design-reference/design-brief.md`) + **نموذج البيانات** (`docs/design-reference/schema.md`) + توجيهات المالك. التصميم مرجع بصري فقط — نبني تطبيق Next.js حقيقي مربوط بقاعدة بيانات.

## ١) نظرة عامة

نظام CRM عقاري لشركة **مشاريع السلطان** (السعودية): يدير العميل من أول اتصال لين البيع والإفراغ، مع مشاريع ووحدات، حجوزات بنكية، تحليلات، ومساعد ذكي. عربي بالكامل، RTL، ثيم أوبسيديان داكن، لهجة سعودية.

## ٢) التقنيات

| الطبقة | الاختيار |
|--------|----------|
| الإطار | Next.js 15 (App Router) + TypeScript |
| التنسيق | Tailwind v4 + shadcn/ui |
| قاعدة البيانات | PostgreSQL عبر Prisma |
| المصادقة | Auth.js (NextAuth v5) — أدوار owner/admin/employee، الموظف برمز PIN |
| المساعد الذكي | route handler يستدعي Anthropic API (مفتاح في `.env` فقط) |
| النشر | Hostinger Node hosting |

## ٣) الهوية البصرية (ثابتة)

ألوان أوبسيديان بالضبط (في `src/app/globals.css`):
خلفية `#0A0A0B` · بطاقات `#141417` · مرتفع `#1C1C20` · حدود `#2A2A30` · ذهبي `#CBA45E` (فاتح `#E2C078`، غامق `#9C7C3C`) · نص `#EDEDF0` (خافت `#9A9AA3`) · نجاح `#2FBF8F` · خطر `#F0685F` · تحذير `#E5B45C` · معلومة `#5B9BD8`.

الخطوط: **IBM Plex Sans Arabic** للواجهة، **Reem Kufi** للشعار. التخطيط Bento (بطاقات بأحجام مختلفة)، شبكة ١٢ عمود، فواصل ٢٤px، لمسة زجاجية خفيفة.

## ٤) الأدوار والصلاحيات

- **owner / admin:** يشوفون كل العملاء والمشاريع والتحليلات، يديرون الموظفين والتوزيع.
- **employee:** يدخل برمز PIN، يشوف ويدير **عملاءه فقط**.
- **التطبيق على الخادم:** كل query/action يتحقق من الجلسة والدور (لا اكتفاء بإخفاء عناصر الواجهة).

## ٥) نموذج البيانات (يتوسّع تدريجيًا)

`User` (role, pinHash) · `Project` · `Unit` · `Lead` (العميل) · `Booking` (الحجز) · `Activity` (التفاعلات) · `Referral` (الإحالات). التفاصيل في `prisma/schema.prisma`. مراحل خط البيع ٩ (كانبان)، أنواع الوحدات، القنوات، البنوك السعودية — كلها enums.

## ٦) الشاشات (من البريف)

1. **لوحة التحكم** — ٦ مؤشرات Bento + ليدات تنتظر أول تواصل (بمؤقّت) + متابعات اليوم + ما بعد البيع/الإحالات + أداء الموظفين + قمع المبيعات.
2. **خط المبيعات (Bookings)** — ٥ مؤشرات + بطاقات حجز بشريط تقدّم البيع وحالة التمويل.
3. **مراحل العملاء (Kanban)** — ٩ أعمدة، بطاقة قابلة للسحب.
4. **كل العملاء (جدول)** — فلاتر متعددة + نقل جماعي.
5. **المشاريع والوحدات** — بطاقات مشاريع + صفوف وحدات.
6. **التحليلات والذكاء** — «اسأل بياناتك» (AI) + تحليلات مالية + مؤشرات احترافية + رسوم.
7. **الموظفين** (للمدير) — بطاقات + توزيع/استيراد.

التفاصيل الكاملة لكل شاشة في `docs/design-reference/design-brief.md`.

## ٨) النشر (Hostinger Node hosting)

ملخّص (التفاصيل في `README.md`):
1. أنشئ PostgreSQL واضبط متغيّرات البيئة (`DATABASE_URL`, `AUTH_SECRET`, `AUTH_URL`, `ANTHROPIC_API_KEY`).
2. `npm install` (يشغّل `prisma generate` تلقائيًا عبر postinstall).
3. `npm run db:deploy` (= `prisma migrate deploy`) لتطبيق الترحيلات على قاعدة الإنتاج.
4. `npm run build` ثم `npm start`.
